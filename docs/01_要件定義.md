# MyTraining 要件定義書

## 1. 目的

日々のトレーニング記録を手軽に入力・蓄積し、最終的に MyOS 側へデータ連携することで、
自分自身のトレーニング傾向・体調の理解に役立てる。

## 2. スコープ

### 対象
- 筋力トレーニング(種目・重量・回数・セット)の記録
- 有酸素運動(種類・時間・距離)の記録
- 体重・コンディション(疲労度・睡眠時間)・メモの記録
- 記録の一覧・編集・削除
- 週間サマリー(トレーニング日数・総ボリューム)
- MyOS 連携用のデータエクスポート(JSON ダウンロード / Webhook 送信)

### 対象外(将来検討)
- 複数ユーザー・アカウント管理
- クラウド同期(コスト無料の制約のため、初期はローカル保存のみ)
- トレーニングメニューの自動提案

## 3. 制約条件

| 項目 | 内容 |
|---|---|
| コスト | ランタイムコスト無料(サーバレス・ローカル保存。外部有料サービス不使用) |
| Web 版 | TypeScript で実装 |
| iOS 版 | App Store 配信可能な形式(Capacitor による同一コードベース) |
| データ保存 | 端末ローカル(localStorage)。バックアップはエクスポート機能で担保 |

> **注意:** App Store への実際の配信には Apple Developer Program(年間 $99)への
> 加入が必要です。これは Apple 側の必須費用であり、アプリ自体の開発・運用コストは無料です。

## 4. 機能要件

| ID | 機能 | 説明 | 優先度 |
|---|---|---|---|
| F-01 | 記録入力 | 日付・筋トレ種目(セットごとの重量/回数)・有酸素・体重・疲労度・睡眠時間・メモを入力し保存 | 必須 |
| F-02 | 記録一覧 | 日付降順で記録を一覧表示 | 必須 |
| F-03 | 記録編集・削除 | 既存記録の修正・削除 | 必須 |
| F-04 | 週間サマリー | 直近7日間のトレーニング日数・総ボリューム(重量×回数)・有酸素時間を表示 | 必須 |
| F-05 | JSON エクスポート | 全記録を MyOS 連携フォーマット(JSON)でダウンロード | 必須 |
| F-06 | Webhook 連携 | 設定した MyOS エンドポイントへ記録を POST 送信 | 推奨 |
| F-07 | 種目名サジェスト | 過去に入力した種目名を候補表示 | 推奨 |

## 5. 非機能要件

| 項目 | 内容 |
|---|---|
| 性能 | 入力・一覧表示は即時(ローカル完結) |
| 可用性 | オフラインで全機能動作(Webhook 送信を除く) |
| データ保全 | エクスポートによる手動バックアップ。保存失敗時はエラー表示 |
| UI | スマートフォン優先のレスポンシブデザイン。日本語 UI |
| 対応環境 | Web: モダンブラウザ / iOS: iOS 14 以上(Capacitor 要件に準拠) |

## 6. 用語

| 用語 | 意味 |
|---|---|
| ボリューム | 重量 × 回数 の合計。筋トレの総負荷量の指標 |
| MyOS | 記録データの最終連携先となる利用者自身のシステム |
