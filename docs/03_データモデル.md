# MyTraining データモデル定義

## 1. エンティティ

### TrainingRecord(1日の記録)

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| id | string | ○ | UUID |
| date | string | ○ | `YYYY-MM-DD` |
| strength | StrengthExercise[] | ○ | 筋トレ種目の配列(空可) |
| cardio | CardioSession[] | ○ | 有酸素の配列(空可) |
| bodyWeightKg | number \| null | - | 体重 (kg) |
| fatigue | 1〜5 \| null | - | 疲労度(1=絶好調, 5=極度の疲労) |
| sleepHours | number \| null | - | 睡眠時間 |
| memo | string | - | 自由記述 |
| createdAt / updatedAt | string | ○ | ISO 8601 |

### StrengthExercise(筋トレ種目)

| フィールド | 型 | 説明 |
|---|---|---|
| name | string | 種目名(例: ベンチプレス) |
| sets | ExerciseSet[] | セットの配列 |

### ExerciseSet

| フィールド | 型 | 説明 |
|---|---|---|
| weightKg | number | 重量 (kg)。自重は 0 |
| reps | number | 回数 |

### CardioSession(有酸素)

| フィールド | 型 | 説明 |
|---|---|---|
| kind | string | 種類(例: ランニング) |
| minutes | number | 時間(分) |
| distanceKm | number \| null | 距離 (km) |

### AppSettings

| フィールド | 型 | 説明 |
|---|---|---|
| myosWebhookUrl | string | MyOS 連携先エンドポイント(空なら未設定) |

## 2. 保存形式(localStorage)

| キー | 内容 |
|---|---|
| `mytraining.records` | `TrainingRecord[]` の JSON |
| `mytraining.settings` | `AppSettings` の JSON |

- 1日1レコードを原則とし、同一日付は上書き確認の上マージではなく別レコードとして許容する
  (実装では日付重複を警告)。
- スキーマ変更時に備え、エクスポート JSON に `schemaVersion` を含める。

## 3. 導出値(保存しない)

| 値 | 計算式 |
|---|---|
| 記録のボリューム | Σ(weightKg × reps) 全セット |
| 週間トレーニング日数 | 直近7日間で strength または cardio が非空の記録数 |
| 週間総ボリューム | 直近7日間のボリューム合計 |
| 週間有酸素時間 | 直近7日間の minutes 合計 |
